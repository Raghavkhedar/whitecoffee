# App-wide Live Sync Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make backend changes reflect in the Android app instantly — an admin suspend/restore blocks/unblocks the app with no re-login, and every screen's data updates live.

**Architecture:** Phase 1 extends the existing `MainViewModel` user-doc snapshot listener to publish an `AccountStatus`, rendered as a full-screen overlay above the nav host in `MainActivity` (auto-lifts on restore). Phase 2 introduces one Firebase-agnostic `listenerFlow` primitive; repositories expose `observeX(): Flow<…>` built on it, ViewModels collect them into their existing `StateFlow`s (the Compose UI is already reactive, so it becomes live for free).

**Tech Stack:** Kotlin, Jetpack Compose, Firestore, Hilt, Coroutines/Flow, JUnit4 (+ kotlinx-coroutines-test, added in Task 6).

## Global Constraints

- Package root: `com.raghav.whitecoffee`.
- Build/type-check: `./gradlew :app:compileDebugKotlin --console=plain` (run from `android/`).
- Unit tests: `./gradlew :app:testDebugUnitTest --console=plain` (run from `android/`).
- Tests are pure JVM JUnit4 with no mocking framework — testable logic must be pure functions over primitives, never over Firebase `DocumentSnapshot`/`Query` types.
- Design system: teal Material 3 via `WcColors`/`ui/theme` — never hardcode colors.
- `deriveAttendanceState` (home-out sticky) and `isEventAllowed` (write-time guard) are pure and already tested in `AttendanceRecordTest` — these tests must stay green through Phase 2.
- Firestore listeners must always be removed on teardown (KEY DECISION #8); `callbackFlow { awaitClose { … } }` and `viewModelScope` handle this.
- Android-only change: do NOT modify the admin portal or Cloud Functions. The `active`/suspension fields are already written by `setUserActive`.

```kotlin
sealed interface AccountStatus {
```

```kotlin
package com.raghav.whitecoffee.data.firestore

import com.google.firebase.firestore.DocumentReference
import com.google.firebase.firestore.DocumentSnapshot
import com.google.firebase.firestore.Query
import com.google.firebase.firestore.QuerySnapshot
import kotlinx.coroutines.channels.awaitClose
import kotlinx.coroutines.flow.Flow
import kotlinx.coroutines.flow.callbackFlow

/** Firebase-agnostic handle so the primitive can be unit-tested without the SDK. */
fun interface Cancelable { fun cancel() }

/**
 * Turns any push-based subscribe/cancel API into a cold Flow. The [subscribe] lambda receives
 * an onNext + onError sink and returns a [Cancelable]; the flow removes the subscription when
 * the collector stops (screen left, ViewModel cleared). This is the only listener-wrapping
 * primitive in the app — the Firestore extensions below are one-liners over it.
 */
fun <T> listenerFlow(
    subscribe: (onNext: (T) -> Unit, onError: (Throwable) -> Unit) -> Cancelable
): Flow<T> = callbackFlow {
    val handle = subscribe({ trySend(it) }, { close(it) })
    awaitClose { handle.cancel() }
}

fun Query.snapshotsAsFlow(): Flow<QuerySnapshot> = listenerFlow { onNext, onError ->
    val reg = addSnapshotListener { snap, err ->
        if (err != null) onError(err) else if (snap != null) onNext(snap)
    }
    Cancelable { reg.remove() }
}

fun DocumentReference.snapshotsAsFlow(): Flow<DocumentSnapshot> = listenerFlow { onNext, onError ->
    val reg = addSnapshotListener { snap, err ->
        if (err != null) onError(err) else if (snap != null) onNext(snap)
    }
    Cancelable { reg.remove() }
}
```
